Github pages + Hexo 搭建个人主页

发表于 2016-12-02 | 分类于 Hexo | 阅读次数

　　尽量写一篇简洁的教程，照顾到很多想要有自己博客的同学。无论你从事什么专业，无论什么年龄，只要你想有一个属于自己的博客，都可以来看，会写的很详细，因为我和你一样，也是小白。
　　其实现在每个软件的官网都把安装步骤写的很详细了，只是即使这样，我们在安装的时候也会出现各种奇奇怪怪的错误，有时候Google也找不到和我们一样错误的人，你会不会觉得天啊，为什么只有我。所以我还是要写一下我安装的步骤，因为在这个过程中，我也遇到无数这样的坑，希望你们不要有坑，但如果有 ，我能帮到你们。
　　先解释一下这几个名词吧，解释的不好勿喷。

名词解释

Github pages

Github Pages 是依托 Github 本身的技术服务经验开发出来的一款产品，解决的是分布式的内容版本管理问题。架设博客是其功能的一部分，正如官网写的Websites for you and your projects.你可以在这里展示你的代码，也可做你想做的事情。

Hexo

Hexo是一个开源的静态博客生成器，用node.js开发，作者是台湾大学生tommy351。官网也有这样一句介绍：A fast, simple & powerful blog framework，因为我也是个小白，也不能说出hexo具体哪里好，但是简单方便是一定的，不然小白我也不能这么快就写个教程出来啊。

node.js

Hexo基于node.js，所以这是安装hexo不可少的环境。
Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境。这是官网的介绍，我觉得我不能更详细了，因为我对前端框架也不熟，在慢慢学习。

申请Github账号

  • 进入Github主页，去申请账号吧，现在github可能被墙，具体代理什么的，看自己爱好吧。

新建仓库

  • 点击右下角的New resposity

  • 在Respository name输入username.github.io.git
    这个地址是固定的，就是你的用户名加上github.io.git这样可以保证每个github账户就只有一个pages地址。不然别人起了这个名字你岂不是用不了。笔者因为已经创建过了，所以塔提示我已经存在了，之后点Create respository就可以了。

  • 仓库建好了，接下来要做的就是本地配置了。

hexo安装

很郁闷，我本来想写的很详细，不知道为啥都推荐你们去官网了，因为官网真的好啊。

  • npm包改为淘宝镜像
    这里需要说明一下，因为国内的网络环境，四面围墙，有时候用npm包会出现慢到卡死的情况，我就出现了这个问题，于是推荐大家改为淘宝镜像。

    这两条命令可以修改为国内镜像，速度快多了。

  • 安装hexo
    直接输入命令

    npm install hexo-cli -g

进行hexo的全局安装

hexo初始设置

  • 建立你在本地的目录

    解释一下这几个命令。

    • init是初始化你的hexo目录。之后你所有的静态页面都是在这个目录下，包括你发的博客和所有资源。
    • generate会生成你的静态页面，包括 你写的md文档。
    • server命令很棒，这个据说也是hexo很受欢迎的一个原因。这个命令可以让你在本地预览hexo生成的网页。在这个命令执行期间，进入 http://localhost:4000 就可以看到你的网页了，这个预览功能很不错，如果你在此时修改了md文件，不需要再次generate，刷新一下网页就立即更改了。退出就和其他指令一样control+c。

  • 修改配置文件，关联到github
    在你的hexo目录下，有一个config.yml文件，打开它，在最后又deploy标签，修改其type为git,respository为你的github仓库地址，博主的配置如下

    deploy:
type: git
repository: http://github.com/wlwlxgg/wlwlxgg.github.io.git
branch: master

    千万要记得，冒号后面有一个空格，博主在配置的时候被这个坑了一天。

  • hexo d —— 将网页发布到Github pages
    终于来了，有人看到现在也不能理解为什么要和github有关系，很多人只以为hexo是重点，我想说一下，hexo只是一个页面生成工具，我们需要服务器把我们的代码发布到互联网上，github pages就是提供给我们的这样一个免费空间，我们通过他把我们的代码展现给互联网。所以当我们在本地用hexo生成了静态页面后，需要把文件发布到我们之前建立的代码仓库，因为这个仓库我们是按照github pages建库的标准建立的，即username.github.io.git，所以当你用hexo把文件同步到仓库后，github会把生成的页面发布到github pages上去，这个时候我们就可以登录 username.github.io 进入到我们的页面了。
